# Documentação do MT 0.3.x Alpha
**Esse documento vale tanto para o MT 0.3 e 0.3.1.**

---

## Scripts
- Apenas dois Scripts, **o Start.mts e Main.mts.**
- O Start.mts serve para configurar regras e threads para o resto do projeto.
- O Main.mts é o script que deverá contero código principal do projeto.
- **Distinção real** entre variáveis locais (apenas para um script) e globais (valem para todos os scripts)

## Multi-Threading
- Multi-Thread cooperativo foi introduzido nessa versão, trazendo a possibilidade de rodar mais de um script ao mesmo tempo.
- São gerenciadas pelos comandos sethread e delthread.
- São a principal novidade dessa versão
- Por enquanto sem suporte a estados de thread.

## Variáveis
- Tipos suportados: número, string, mas não boolean.
- Comando: Variablet (set, change) (global, local) (nome) (valor)
- Sobrescrita permitida

## Comandos disponíveis
- Variablet (set, change) (global, local) (nome) (valor)
- Print (coisa) (sem aspas, sem espaço no texto)
- Repeat
- Setflag (nome) (vezes em que pode ser chamada)
quando uma flag é chamada, o executor vai até a flag e continua a partir dela, se a flag estiver inválida, nada será feito.
- Gotoflag (nome da flag)
- setrule (regra) (valor)
- sethread (thread) (script para rodar)
- delthread (thread)
- wait.s (segundos) **(Espera uma quantidade determinada de segundos, números decimais são aceitos.)**
- wait.kp (tecla) **(Espera uam tecla especificada ser pressionada antes de continuar.)**

## Regras do Executor
- **Loudscript:** se definido para 1, retorna mais informações sobre a execução, útil para debug.
- **MaxThreads:** se definida para qualquer valor, não deixa existir mais threads do que aquele valor, por exemplo, se MaxThreads estiver definido para *16*, não existirão mais que *16* threads. Solicitações de criação de threads se o limite estivr atingido serão recusadas internamente.
- **SpeedCap:** Defina essa regra para *0* e o MT irá rodar sem limite! Ou seja, ele vai rodar absurdamente rápido, se você não quer isso, defina qualquer número no Speedcap para limitar a velocidade de execução do MT. **(NÃO COLOQUE VALORES NEGATIVOS!)**
- **ConsoleSize:** Se definido para *auto*, o tamanho do console irá mudar conforme o tamanho da janela, se não quiser isso, pode colocar qualquer número para mudar o tamanho do console para um tamanho fixo.

## ShedExec e Núcleo

### O novo executor do MT se chama **ShedExec**, Reunindo Executor e Sheduler num mesmo ponto central.
- Roda bytecode feito pelo parser e roda comandos a patir dele..
- Gerencia e cria Threads.
- É um sistema unificado.

### Parser
- Responsável por transformar código legível para bytecode que o ShedExec entende.
- Contém verificação básica de projeto corrompido.

### Save Engine
- Responsável por administrar os dois scripts.
- Carrega um de cada vez na hora de editar.

## Limitações Conhecidas
- Sem suporte a mais de dois scripts
- Sem suporte a salvamento de projetos
- Bug crítico no MT 0.3 que deixa o sistema de flags rodarem bytecode, trazendo sérios problemas e até crashes. **Foi corrigido na 0.3.1.**
